# VPSサーバーへのアップデート（更新）手順

本アプリケーション (`salon-reservation-app`) を VPS サーバーにデプロイした後に、ソースコードの修正を反映させるための標準的な手順です。

## 前提条件
*   GitHub にソースコードがプッシュされていること。
*   VPS サーバーに SSH で接続できること。
*   サーバー上で `pm2` などのプロセス管理ツールが使用されていることを想定しています。

---

## ステップ 1: 手元のPCで修正をGitHubに反映
全ての修正が終わったら、GitHub に最新のコードを送信します。

```bash
git add .
git commit -m "修正内容のメモ（例：予約カードの不具合修正）"
git push origin main
```

---

## ステップ 2: VPSサーバーにログインしてコードを更新
SSH でサーバーに接続し、最新のコードを取り込みます。

1.  **サーバーにログイン**
    ```bash
    ssh ユーザー名@サーバーのIPアドレス
    ```

2.  **プロジェクトディレクトリに移動**
    ```bash
    cd /path/to/salon-reservation-app  # 実際のパスに書き換えてください
    ```

3.  **最新コードをプル**
    ```bash
    git pull origin main
    ```

---

## ステップ 3: 依存関係とデータベースの更新
ライブラリの追加やデータベース構造（Prisma）の変更を反映させます。

1.  **ライブラリのインストール**（必要な場合のみ）
    ```bash
    npm install
    ```

2.  **Prisma クライアントの生成とスキーマ反映**
    > [!IMPORTANT]
    > `db push` は SQLite を使用している場合に直接スキーマを同期します。本番環境でのデータ消失には十分注意し、必要に応じて事前に `dev.db` をバックアップしてください。

    ```bash
    npx prisma generate
    npx prisma db push
    ```

---

## ステップ 4: アプリケーションの再ビルドと再起動
最新コードで実行ファイルを生成し、動いているプログラムを入れ替えます。

1.  **ビルドの実行**
    ```bash
    npm run build
    ```

2.  **サービスの再起動 (PM2 の場合)**
    ```bash
    pm2 restart salon-app  # 「salon-app」は設定したプロセス名に合わせてください
    ```

---

## 補足：トラブルシューティング
*   **環境変数の変更**: `.env` ファイルを修正した場合は、必ずビルド (`npm run build`) と再起動を行ってください。
*   **データベースのバックアップ**: 更新作業の前に `cp dev.db dev.db.bak` などでバックアップを取ることを強くお勧めします。
